The Cookbook is task-first. Where the [Get Started](/get-started/index) tab teaches the mental model and the [API Reference](/api-reference/index) documents every available endpoint, the Cookbook hands you a working result you can paste into a terminal or an editor and adapt. Every recipe states a concrete goal, lists what you need first, walks through runnable code step by step, explains what the code does, and points to a sensible next step.

All recipes assume the same two facts about Nito. It is OpenAI-compatible, so you talk to `POST /v1/chat/completions` and `GET /v1/models` with a single organization key sent as `Authorization: Bearer <z_sk_ key>`. Model IDs come from `GET /v1/models`, and privacy is selected per call by a suffix on the model ID (`:confidential` or `:encrypted`). If those two ideas are new, read [How Nito works](/get-started/how-nito-works) before you start.

## Getting started recipes

These are the on-ramps. Start here if you are new, or jump straight to the migration recipe that matches the provider you are moving from.

- [**Quickstart recipe**](/cookbook/getting-started/quickstart-recipe) the shortest end-to-end path: one key, one request, one response, with curl and Python side by side.
- [**Migrate from OpenAI**](/cookbook/getting-started/migrate-from-openai) keep your OpenAI SDK code and repoint it at Nito. Chat Completions carries over; the Responses API does not.
- [**Migrate from Anthropic**](/cookbook/getting-started/migrate-from-anthropic) move off the Anthropic Messages API onto Nito's OpenAI-compatible Chat Completions surface.
- [**Migrate from OpenRouter**](/cookbook/getting-started/migrate-from-openrouter) swap the base URL and key, then map model names to Nito catalog IDs.
- [**Migrate from other gateways**](/cookbook/getting-started/migrate-from-others) the same base-URL swap for Together, Fireworks, Replicate, Perplexity, and similar OpenAI-compatible providers.
- [**Build a first chat app**](/cookbook/getting-started/build-first-chat-app) a minimal but complete app that sends a conversation and prints the reply.
- [**Build a streaming chat UI**](/cookbook/getting-started/build-streaming-chat-ui) render tokens as they arrive over Server-Sent Events, with a framework-light front-end sketch.

## How to read a recipe

Each page follows the same shape so you can scan it fast:

1. **Goal** one sentence on what you will have when you finish.
2. **Prerequisites** the key, the base URL, and any tools you need installed.
3. **Steps** numbered, each with runnable code you can copy directly.
4. **Walkthrough** a short explanation of why the code is shaped the way it is.
5. **Next steps and Related** where to go deeper.

Wherever a recipe shows a full, concrete example, it uses `x-ai/grok-4.5` as the example model and `https://nito.dev.zprotocol.org` as the base URL placeholder. Both are stand-ins: fetch the live catalog with `GET /v1/models` and substitute your real gateway URL.

## What is not here yet

Recipes that build autonomous agents (tool loops, planning, multi-step task execution) are deferred. Those patterns are best expressed through the Agent SDK, which lives in a separate repository and is documented on its own track. When that lands, the agent recipes will appear as their own group. For now, the Cookbook covers the request-and-response surface you can reach directly over HTTP.
