## Goal

Move existing OpenAI SDK code onto Nito without rewriting your application. When you finish, the same Chat Completions calls run against Nito's catalog.

## Prerequisites

- Code that already uses the OpenAI SDK (Python or TypeScript) or hits `POST /v1/chat/completions` directly.
- A Nito API key starting with `z_sk_`. See [Authentication](/get-started/authentication).
- A chat model ID from `GET /v1/models`. This recipe uses `grok-4.3` as an example.

## The three changes

Migration is a base-URL swap. You change exactly three things and leave the rest of your code alone:

1. **Base URL** points at the Nito gateway plus `/v1`.
2. **API key** becomes your `z_sk_` key.
3. **Model ID** becomes a Nito catalog ID, not an OpenAI name like `gpt-4o`.

## Step 1: Repoint the client

<CodeGroup>

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="z_sk_your_key_here",              # was your OpenAI key
    base_url="https://YOUR-NITO-BASE-URL/v1",  # was https://api.openai.com/v1
)

resp = client.chat.completions.create(
    model="grok-4.3",                          # was e.g. gpt-4o
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
)

print(resp.choices[0].message.content)
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "z_sk_your_key_here",              // was your OpenAI key
  baseURL: "https://YOUR-NITO-BASE-URL/v1",  // was https://api.openai.com/v1
});

const resp = await client.chat.completions.create({
  model: "grok-4.3",                         // was e.g. gpt-4o
  messages: [{ role: "user", content: "Say hello in one short sentence." }],
});

console.log(resp.choices[0].message.content);
```

## Step 2: Map your model names

OpenAI model names do not exist on Nito. Fetch the live catalog and pick equivalents by capability:

```bash
curl "https://YOUR-NITO-BASE-URL/v1/models" \
  -H "Authorization: Bearer z_sk_your_key_here"
```

Select IDs at runtime from this list rather than pasting a name into your source, because the catalog is discovered from providers and changes over time.

## Walkthrough

Nito implements the OpenAI Chat Completions protocol, so `messages`, `temperature`, `top_p`, `max_tokens`, `tools`, `tool_choice`, streaming, and the `chat.completion` response shape all carry over unchanged. The SDK appends `/chat/completions` to your base URL, which is why the base URL must end in `/v1`.

To select a privacy mode, add a suffix to the model ID (`grok-4.3:confidential`). No request-body field controls privacy; it is purely the model string. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## What does not carry over

- **This recipe uses Chat Completions.** Chat Completions is the primary, fully supported surface. Nito now also serves an OpenAI Responses-compatible endpoint at `POST /v1/responses`; if your code calls `client.responses.create(...)`, you can either target that endpoint or port the call to `client.chat.completions.create(...)`.
- **OpenAI model names.** `gpt-4o` and friends are not catalog IDs. Always substitute a Nito ID.
- **Encrypted and streaming together.** `:encrypted` does not stream. Use `:confidential` if you need streaming with attestation.

Do not assume OpenAI-specific behaviors that are not part of the Chat Completions contract. If a feature is not documented for Nito, treat it as unverified rather than importing OpenAI's semantics.
