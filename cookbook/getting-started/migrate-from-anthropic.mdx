## Goal

Move an application built on the Anthropic Messages API onto Nito. This recipe routes your calls through Nito's OpenAI-compatible Chat Completions surface, a supported and stable path. Nito now also serves an Anthropic Messages-compatible endpoint at `POST /v1/messages`, but this recipe uses the Chat Completions route.

## Prerequisites

- Code that currently calls the Anthropic Messages API (`POST /v1/messages`) or uses the Anthropic SDK.
- A Nito API key starting with `z_sk_`. See [Authentication](/get-started/authentication).
- A chat model ID from `GET /v1/models`. This recipe uses `grok-4.3` as an example.

## The key difference

This recipe translates the request shape from Messages to Chat Completions rather than doing a base-URL-only swap. The translation is mechanical, and the easiest path is to use the OpenAI SDK against Nito. Nito now also serves an Anthropic Messages-compatible endpoint at `POST /v1/messages` and `POST /v1/messages/count_tokens`; the Chat Completions translation below remains a fully supported option.

## Step 1: Translate the request shape

The Messages API and Chat Completions differ in two visible ways. The system prompt moves from a top-level `system` field into a message with `role: "system"`, and the response text is read from `choices[0].message.content` instead of `content[0].text`.

<CodeGroup>

```python Python (OpenAI SDK against Nito)
from openai import OpenAI

client = OpenAI(
    api_key="z_sk_your_key_here",
    base_url="https://nito.dev.zprotocol.org/v1",
)

resp = client.chat.completions.create(
    model="grok-4.3",  # a Nito catalog ID from GET /v1/models
    messages=[
        {"role": "system", "content": "You are a concise assistant."},
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
    max_tokens=256,
)

print(resp.choices[0].message.content)
```

```bash curl
curl "https://nito.dev.zprotocol.org/v1/chat/completions" \
  -H "Authorization: Bearer z_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.3",
    "messages": [
      { "role": "system", "content": "You are a concise assistant." },
      { "role": "user", "content": "Say hello in one short sentence." }
    ],
    "max_tokens": 256
  }'
```

## Step 2: Map your model names

Anthropic model names are not Nito catalog IDs. Fetch the catalog and select by capability:

```bash
curl "https://nito.dev.zprotocol.org/v1/models" \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Translation reference

| Anthropic Messages | Nito Chat Completions |
| --- | --- |
| top-level `system` string | a message with `role: "system"` |
| `messages: [{role, content}]` | `messages: [{role, content}]` (same) |
| `max_tokens` (required) | `max_tokens` (optional) |
| `stream: true` (SSE events) | `stream: true` (SSE chunks, `data: [DONE]`) |
| `content[0].text` in response | `choices[0].message.content` |

## Walkthrough

Once the request is in Chat Completions shape, everything else is standard Nito. Streaming uses `stream: true` and emits SSE chunks ending with `data: [DONE]`. Privacy is chosen by a model-ID suffix (`grok-4.3:confidential`), not a request field. Only `:confidential` and `:encrypted` are valid, and `:encrypted` does not stream.

Do not carry Anthropic-specific behaviors over as if they were Nito's. Features that are part of the Anthropic API but not documented for Nito's Chat Completions surface should be treated as unverified rather than assumed to work.

## What does not carry over

- **This recipe uses Chat Completions.** It routes through `/v1/chat/completions`. Nito now also serves an Anthropic Messages-compatible endpoint at `/v1/messages` and `/v1/messages/count_tokens` if you prefer the native shape.
- **Anthropic model names.** Substitute Nito catalog IDs.
