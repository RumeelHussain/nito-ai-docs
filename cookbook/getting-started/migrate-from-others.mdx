## Goal

Move an application off another OpenAI-compatible provider (Together, Fireworks, Replicate, Perplexity, and similar) onto Nito with a base-URL-and-key swap plus a model-name remap.

## Prerequisites

- Code that calls an OpenAI-compatible endpoint on your current provider (the OpenAI SDK, or direct `POST /chat/completions`).
- A Nito API key starting with `z_sk_`. See [Authentication](/get-started/authentication).
- A chat model ID from `GET /v1/models`. This recipe uses `grok-4.3` as an example.

## The pattern

If your current provider exposes an OpenAI-compatible Chat Completions endpoint, migration is the same three edits every time:

1. **Base URL** points at the Nito gateway plus `/v1`.
2. **API key** becomes your `z_sk_` key.
3. **Model ID** becomes a Nito catalog ID.

## Step 1: Repoint the client

<CodeGroup>

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="z_sk_your_key_here",              # was your provider's key
    base_url="https://nito.dev.zprotocol.org/v1",  # was your provider's /v1 base URL
)

resp = client.chat.completions.create(
    model="grok-4.3",                          # a Nito catalog ID from GET /v1/models
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
)

print(resp.choices[0].message.content)
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "z_sk_your_key_here",              // was your provider's key
  baseURL: "https://nito.dev.zprotocol.org/v1",  // was your provider's /v1 base URL
});

const resp = await client.chat.completions.create({
  model: "grok-4.3",                         // a Nito catalog ID
  messages: [{ role: "user", content: "Say hello in one short sentence." }],
});

console.log(resp.choices[0].message.content);
```

## Step 2: Map your model names

Each provider names models its own way. Fetch Nito's live catalog and select equivalents by capability:

```bash
curl "https://nito.dev.zprotocol.org/v1/models" \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Provider notes

The swap works the same for each, but the model-name step differs:

| Coming from | What to change |
| --- | --- |
| Together | Base URL, key, and remap Together model slugs to Nito IDs. |
| Fireworks | Base URL, key, and remap `accounts/.../models/...` slugs to Nito IDs. |
| Replicate | Base URL and key; use the OpenAI-compatible Chat Completions call, and remap model refs to Nito IDs. |
| Perplexity | Base URL, key, and remap Perplexity model names to Nito IDs. |

## Walkthrough

All of these speak the OpenAI Chat Completions protocol, so `messages`, sampling parameters, `tools`, streaming, and the response shape carry over. Privacy on Nito is a model-ID suffix (`grok-4.3:confidential`); only `:confidential` and `:encrypted` are valid, and `:encrypted` does not stream.

Do not carry any provider-specific behavior over as if it were Nito's. Custom request fields, routing preferences, search parameters, or response extensions that are specific to your old provider are not part of Nito's contract. If a capability is not documented for Nito, treat it as unverified. Nito's supported surface is Chat Completions, embeddings, image generation, and the models catalog, all reachable under `/v1`.
