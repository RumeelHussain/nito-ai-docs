## Goal

Build a small but complete chat app that keeps a conversation going across turns and prints each reply. When you finish you will understand how to carry message history, which is the core of every chat interface.

## Prerequisites

- A Nito API key starting with `z_sk_`. See [Authentication](/get-started/authentication).
- Python 3 with the `openai` package installed (`pip install openai`), or curl for the single-turn version.
- A chat model ID from `GET /v1/models`. This recipe uses `grok-4.3` as an example.

## Step 1: A single call with curl

Start by confirming a bare request works:

```curl
curl "https://nito.dev.zprotocol.org/v1/chat/completions" \
  -H "Authorization: Bearer z_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.3",
    "messages": [
      { "role": "system", "content": "You are a helpful, concise assistant." },
      { "role": "user", "content": "What is a gateway, in one sentence?" }
    ]
  }'
```

## Step 2: A multi-turn loop in Python

A chat app is just a growing list of messages. Each turn you append the user's line, send the whole list, then append the assistant's reply so the model has context next time:

```python
from openai import OpenAI
import os

client = OpenAI(
    api_key=os.environ["NITO_API_KEY"],
    base_url=os.environ["NITO_BASE_URL"] + "/v1",  # base URL must end in /v1
)

# The conversation history. The model sees this whole list every turn.
messages = [
    {"role": "system", "content": "You are a helpful, concise assistant."},
]

print("Chat with the model. Type 'exit' to quit.\n")

while True:
    user_input = input("you> ").strip()
    if user_input.lower() in {"exit", "quit"}:
        break

    messages.append({"role": "user", "content": user_input})

    resp = client.chat.completions.create(
        model="grok-4.3",  # a Nito catalog ID from GET /v1/models
        messages=messages,
    )

    reply = resp.choices[0].message.content
    print(f"bot> {reply}\n")

    # Persist the reply so the next turn has full context.
    messages.append({"role": "assistant", "content": reply})
```

Set your environment first:

```javascript
export NITO_API_KEY="z_sk_your_key_here"
export NITO_BASE_URL="https://nito.dev.zprotocol.org"
```

## Walkthrough

The whole app rests on one idea: the API is stateless, so the conversation lives in your `messages` list, not on the server. Nito keeps metadata only by default and does not retain your conversation, so it is your job to send the history each turn. That is why the loop appends both the user's input and the assistant's reply before looping again. Drop the append of the assistant's reply and the model forgets everything you said.

The `system` message at the top sets behavior once and stays for the whole session. `resp.choices[0].message.content` is the reply text, and `finish_reason` (not printed here) would tell you if the model stopped early because it hit `max_tokens` (`length`) rather than finishing (`stop`).

To send a turn through a stronger privacy mode, change the model to `grok-4.3:confidential`. Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the mode.

## Watch out for growth

The message list grows every turn, and long conversations cost more tokens and can hit context limits. In a real app you would trim or summarize old turns. For this recipe, a short session is fine.
