A Discord bot has two halves. The Discord half connects over the gateway, receives message events, and posts replies. The Nito half turns the user's text into an answer. This recipe owns the Nito half and shows the Discord half only as a representative shape, because Discord's API and client libraries change over time.

## The Nito call

The bot's brain is one call to Nito's OpenAI-compatible Chat Completions endpoint. Nito is OpenAI-compatible, so you can use the OpenAI SDK and change only the base URL, key, and model.

```python
from openai import OpenAI

nito = OpenAI(
    api_key="z_sk_your_key_here",              # your Nito API key, keep it server-side
    base_url="https://nito.dev.zprotocol.org/v1",  # Nito's OpenAI-compatible base URL
)

def answer(user_text: str) -> str:
    resp = nito.chat.completions.create(
        model="x-ai/grok-4.5",                      # a Nito catalog ID, not an OpenAI name
        messages=[
            {"role": "system", "content": "You are a helpful Discord assistant."},
            {"role": "user", "content": user_text},
        ],
    )
    return resp.choices[0].message.content
```

The base URL must end in `/v1`, because the SDK appends `/chat/completions` itself. Use a Nito catalog ID for `model`, not an OpenAI name like `gpt-4o`. Fetch the live list and pick one:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Wiring it to Discord

The representative structure is: an event fires when a message is posted, you pass its text to `answer()`, then reply in the channel. The library names and decorators below are illustrative only.

```python
# Representative only, verify against current Discord docs.
async def on_message(message):
    if message.author.is_bot:
        return
    try:
        reply = answer(message.content)
    except Exception:
        reply = "Sorry, I could not answer that right now."
    await message.channel.send(reply)
```

Keep the Nito key in a server-side environment variable. A Discord bot runs on your own host, so the `z_sk_` key never reaches a client. Never ship it to the browser or a desktop client.

## Streaming or not

Discord posts whole messages, so a bot normally uses a non-streaming call and sends the reply once it is complete. If you want a "typing" effect, you can stream from Nito (`stream=True`) and edit the message as chunks arrive, respecting Discord's edit rate limits. Note that `:encrypted` does not stream, so combine streaming only with an ordinary or `:confidential` model.

## Choosing a privacy tier

Privacy is selected by the model ID. To run a request in a TEE with a verifiable attestation, append `:confidential`:

```python
model="openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier. Use `GET /v1/models` to find attestation-capable models. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Handling errors

Nito returns a JSON error envelope `{"error": {"message", "code", "param?}}` with no `type` field. Map the status to a graceful channel reply:

- **429**: honor `Retry-After` and back off. This covers both rate limits and quota exhaustion.
- **402** or quota: post a short "temporarily unavailable" note and alert your team.
- **400**: usually a bad model ID or malformed message. Confirm you used a catalog ID.

Wrap the Nito call in a try/except so one failed reply does not tear down your event loop.
