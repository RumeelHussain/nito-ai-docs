Codex is an OpenAI-oriented coding tool. You can route it through Nito because Nito exposes an OpenAI-compatible API. You override its base URL, key, and model rather than selecting a named "Nito" provider.

This page is a recipe pointer. The full, maintained setup lives in the [Codex editor integration](/sdks/editor-integrations/codex).

## The three values

Wherever Codex accepts an OpenAI-compatible base URL and key, supply:

- **Base URL:** `https://nito.dev.zprotocol.org/v1`. The path must end in `/v1`, because OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `x-ai/grok-4.5`. Do not use an OpenAI name like `gpt-4o`.

Fetch a real model ID from the live catalog:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Use Chat Completions

This recipe uses OpenAI **Chat Completions** at `POST /v1/chat/completions`, the primary supported surface. Nito now also serves an OpenAI **Responses-compatible** endpoint at `POST /v1/responses` (and `/v1/responses/compact`). Newer OpenAI-oriented tools sometimes default to the Responses API; confirm which surface your Codex version targets, and use Chat Completions where it is available.

## Notes

- Privacy is selected by the model ID. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `openai/gpt-oss-120b:confidential`. `:encrypted` does not stream and needs extra headers, so it is generally not usable from a coding tool's request loop. See [Per-call privacy modes](/privacy/per-call-privacy-modes).
- For the current settings path and troubleshooting, follow the [Codex editor integration](/sdks/editor-integrations/codex) page.
