Claude Code is an Anthropic-oriented CLI. This recipe routes it through Nito's **OpenAI-compatible** surface, which is a supported, working path. Nito now also serves an Anthropic Messages-compatible endpoint at `POST /v1/messages`; full drop-in guidance for it is documented in a later pass.

This page is a recipe pointer. The full, maintained setup lives in the [Claude Code editor integration](/sdks/editor-integrations/claude-code).

## The three values

Wherever Claude Code accepts an OpenAI-compatible base URL and key, supply:

- **Base URL:** `https://YOUR-NITO-BASE-URL/v1`. The path must end in `/v1`, because OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `grok-4.3`. Do not use an Anthropic name like `claude-3-5-sonnet` or an OpenAI name like `gpt-4o`.

Fetch a real model ID from the live catalog:

```bash
curl https://YOUR-NITO-BASE-URL/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Which surface to use

This recipe uses OpenAI Chat Completions at `POST /v1/chat/completions`. Nito now also serves an Anthropic Messages-compatible endpoint at `POST /v1/messages` (and `POST /v1/messages/count_tokens`). Confirm in the official Claude Code docs which surface your version targets, and prefer the OpenAI-compatible provider setting where it is available.

## Notes

- Privacy is selected by the model ID. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:confidential`. `:encrypted` does not stream and needs extra headers, so it is generally not usable from an interactive CLI. See [Per-call privacy modes](/privacy/per-call-privacy-modes).
- For the current settings path and troubleshooting, follow the [Claude Code editor integration](/sdks/editor-integrations/claude-code) page.
