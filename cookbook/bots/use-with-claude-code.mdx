Claude Code is an **Anthropic client** — it speaks the Anthropic Messages API (`POST /v1/messages`), not an OpenAI-compatible one. Route it through Nito by overriding its Anthropic base URL and key. There is no OpenAI-style (`/v1/chat/completions`) configuration for Claude Code.

This page is a recipe pointer. The full, maintained setup lives in the [Claude Code editor integration](/sdks/editor-integrations/claude-code).

## The three values

Claude Code reads these from environment variables:

- **`ANTHROPIC_BASE_URL`:** `https://nito.dev.zprotocol.org` — the **host root, without `/v1`**. Claude Code appends `/v1/messages` itself.
- **`ANTHROPIC_AUTH_TOKEN`:** your Nito key (`z_sk_...`), sent as a bearer token. (Use `ANTHROPIC_AUTH_TOKEN`, not `ANTHROPIC_API_KEY`.)
- **`ANTHROPIC_MODEL`:** a Nito catalog ID the Anthropic facade serves, such as `anthropic/claude-sonnet-5`. Not every model works here — `x-ai/grok-4.5` is rejected on `/v1/messages`; Claude- and GPT-family catalog IDs work.

Fetch a real model ID from the live catalog:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Notes

- Privacy is selected by the model ID. Use a `:confidential` catalog ID for a TEE-backed run with a verifiable attestation, for example `openai/gpt-oss-120b:confidential` (which the Anthropic facade serves). `:encrypted` does not stream and needs extra headers, so it is generally not usable from an interactive CLI. See [Per-call privacy modes](/privacy/per-call-privacy-modes).
- For the current settings path and troubleshooting, follow the [Claude Code editor integration](/sdks/editor-integrations/claude-code) page.
