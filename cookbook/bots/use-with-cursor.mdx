Cursor can use Nito as its model provider because Nito exposes an OpenAI-compatible API. You do not select a named "Nito" provider. You use Cursor's OpenAI-compatible override and point it at Nito.

This page is a recipe pointer. The full, maintained setup lives in the [Cursor editor integration](/sdks/editor-integrations/cursor).

## The three values

Wherever Cursor asks for OpenAI-compatible connection details, supply:

- **Base URL:** `https://nito.dev.zprotocol.org/v1`. The path must end in `/v1`, since OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `grok-4.3`. Do not enter an OpenAI name like `gpt-4o`.

Fetch a real model ID from the live catalog:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Notes

- Privacy is selected by the model ID. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:confidential`. See [Per-call privacy modes](/privacy/per-call-privacy-modes).
- Nito serves OpenAI Chat Completions at `POST /v1/chat/completions`, which is what this recipe uses. Nito now also serves an OpenAI Responses-compatible endpoint at `POST /v1/responses`; where Cursor works best with Chat Completions, leave any "use Responses API" option off.
- For the current settings path, model picker behavior, and troubleshooting, follow the [Cursor editor integration](/sdks/editor-integrations/cursor) page.
