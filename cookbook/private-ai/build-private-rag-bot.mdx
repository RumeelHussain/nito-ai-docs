Retrieval-augmented generation (RAG) has two model calls in its hot path: **embeddings** to find relevant context, and **chat** to answer using that context. On Nito, both surfaces take the same privacy selection, so you can keep the whole pipeline at a private or attested tier. This recipe wires up a minimal private RAG bot.

## The two calls, both private

Privacy is chosen by model ID on every call. That includes embeddings: `POST /v1/embeddings` accepts a `model`, and the same tier rules apply. Choosing a private-tier embedding model, or an attestation-capable one with `:confidential`, keeps retrieval under the same privacy posture as your chat answering step.

Fetch `GET /v1/models?capability=embeddings` (via `GET /api/v1/model-groups?capability=embeddings`) to pick an embedding model, and `GET /v1/models` for a chat model. The example IDs below are placeholders.

## Step 1: Embed your documents

`POST /v1/embeddings` takes `input` (a string or an array of strings) and returns a list of vectors. Limits to respect: at most **96 inputs** per request, each at most **32 KiB**, and `dimensions` (if you set it) bounded 1–3072. The response is `{object: "list", data: [{embedding: [...]}], usage}`. Note `encoding_format` supports `""` or `float` only; `base64` is not supported.

```python
import os, requests

BASE_URL = "https://YOUR-NITO-BASE-URL"       # TODO: confirm base URL
EMBED_MODEL = "EMBED_MODEL_ID:confidential"             # TODO: attestation-capable or private-tier embedding model
API_KEY = os.environ["NITO_API_KEY"]

def embed(texts):
    resp = requests.post(
        f"{BASE_URL}/v1/embeddings",
        headers={"Authorization": f"Bearer {API_KEY}"},
        json={"model": EMBED_MODEL, "input": texts},  # <= 96 items, <= 32 KiB each
        timeout=120,
    )
    resp.raise_for_status()
    return [d["embedding"] for d in resp.json()["data"]]

# Chunk your corpus, then embed in batches of <= 96.
chunks = ["...doc chunk 1...", "...doc chunk 2..."]
vectors = embed(chunks)
```

Store each `(chunk, vector)` pair in whatever vector index you already use. The gateway is metadata-only and does not persist your inputs or the vectors, so your index is yours to host and secure.

> **Note:** per-model output dimensions are not encoded in the gateway; the provider returns the vector length at runtime. Do not hardcode a dimension; read it from the first response.

## Step 2: Retrieve and answer

At query time, embed the question with the **same** embedding model, find the nearest chunks in your index, and pass them to a private or attested chat model as context.

```python
CHAT_MODEL = "CHAT_MODEL_ID:confidential"   # TODO: attestation-capable or private-tier chat model

def answer(question, top_chunks):
    context = "\n\n".join(top_chunks)
    messages = [
        {"role": "system", "content": "Answer only from the provided context. If unsure, say so."},
        {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"},
    ]
    resp = requests.post(
        f"{BASE_URL}/v1/chat/completions",
        headers={"Authorization": f"Bearer {API_KEY}"},
        json={"model": CHAT_MODEL, "messages": messages},
        timeout=120,
    )
    resp.raise_for_status()
    return resp.json()["choices"][0]["message"]["content"]

q_vec = embed([question])[0]
top = nearest(q_vec, k=4)           # your vector index does this
print(answer(question, top))
```

## Keeping the pipeline private end to end

- **Same tier on both calls.** If retrieval runs at a private tier but answering runs on an ordinary model, your privacy posture is only as strong as the weaker call. Match the tiers.
- **Embeddings inherit the same privacy options.** The `:confidential` suffix and private-tier selection work the same way on `/v1/embeddings` as on chat, so there is no separate mechanism to learn.
- **Your index is outside the gateway.** The gateway does not store chunks or vectors. Retention, encryption, and access control on the vector store are your responsibility.
- **Attest if you used `:confidential`.** For an attested chat model, you can fetch and verify the attestation separately. See [TEE-attested call and verify nonce](/cookbook/private-ai/tee-attested-call-verify-nonce).
