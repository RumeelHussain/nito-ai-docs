---
title: Error codes
description: The error families Nito returns in the code field, what each means, and how to respond.
---

The `code` field in the [error envelope](/api-reference/errors-and-rate-limits/error-envelope) carries the error family. These are the only families the gateway emits, with `internal_error` as the fallback. Branch your handling on these values.

## Codes

| Code | Meaning |
|---|---|
| `invalid_request` | The request was malformed, failed validation, or used an unsupported feature/attachment. |
| `model_unavailable` | The requested model could not be resolved or is not available. The `param` is `model`. |
| `unauthorized` | The credential is missing, malformed, revoked, or expired. |
| `forbidden` | Authenticated, but not permitted to perform this action. |
| `insufficient_quota` | The org/key is over a spend or usage quota. |
| `payment_required` | Payment is required before the request can proceed. |
| `provider_error` | An upstream provider returned an error. |
| `rate_limited` | A rate limit was exceeded. |
| `unavailable` | The service or upstream is temporarily unavailable. |
| `internal_error` | Fallback for an unexpected/unclassified error. |

## How to respond

| Code | Recommended action |
|---|---|
| `invalid_request` | Fix the request. Do not retry unchanged. Check `message` and `param`. |
| `model_unavailable` | Pick another model. See [List models](/api-reference/models-catalog/list-models). |
| `unauthorized` | Check the `z_sk_` key and `Authorization` header. Do not retry unchanged. |
| `forbidden` | Check org role/permissions. Do not retry unchanged. |
| `insufficient_quota` | Wait or raise quota, then retry after `Retry-After`. |
| `payment_required` | Add funds/payment, then retry. |
| `provider_error` | Transient. Retry with backoff. See [Retry behavior](/api-reference/errors-and-rate-limits/retry-behavior). |
| `rate_limited` | Retry after `Retry-After`. See [Rate limits](/api-reference/errors-and-rate-limits/rate-limits). |
| `unavailable` | Transient. Retry with backoff. |
| `internal_error` | Retry with backoff; if it persists, contact support with the `X-Request-ID`. |

For the HTTP status each code maps to, see [Status mapping](/api-reference/errors-and-rate-limits/status-mapping). For which are safe to retry, see [Retry behavior](/api-reference/errors-and-rate-limits/retry-behavior).
