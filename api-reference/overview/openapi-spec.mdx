## What this means until then

Until the spec lands, the request and response schemas documented across this API Reference are **derived directly from the gateway's Go source** at the analyzed build, not from a published contract. They are accurate to the code that was mined, but:

- There is no file you can feed to a code generator or import into Postman or an HTTP client.
- There is no automated client SDK generation path.
- Schemas here are hand-derived, so treat the per-endpoint reference pages as the authoritative description in the meantime.

## How to integrate now

Build against the documented endpoints directly. Send strict JSON (unknown fields are rejected), read the `object` field on responses to discriminate shapes, and branch error handling on `error.code`. The OpenAI-compatible `/v1` surface means existing OpenAI-style clients work by pointing the base URL at Nito.

Once `docs/planning/openapi.yaml` is published, this page will link to it and the generated tooling it enables.
