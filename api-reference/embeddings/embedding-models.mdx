description: How to discover embedding models and what the default is.
---

The embeddings endpoint accepts an optional `model`. When you omit it, the gateway uses its default embedding model.

## Discover available models

The model catalog is discovered from providers at runtime, not published as a static list. To see the embedding models your deployment actually serves, list models and filter by type:

```bash
curl "https://nito.dev.zprotocol.org/v1/models?type=embedding" \
  -H "Authorization: Bearer z_sk_your_key_here"
```

Pass the `model` value from a returned model card into the [Create embeddings](/api-reference/embeddings/create-embeddings) request.

## The default model

When `model` is omitted, the request falls back to the gateway's default embedding model. The native default id is `Qwen/Qwen3-Embedding-0.6B`, served by the NEAR and Phala providers.

There is no hardcoded, publishable list of embedding models in the gateway; the catalog is synced from provider APIs. Treat the `GET /v1/models?type=embedding` listing as the source of truth for what is callable right now.
