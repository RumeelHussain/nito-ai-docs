---
title: List models
description: List the chat and image models available on Nito, with their capabilities, context window, and pricing.
---

`GET /v1/models` (OpenAI-compat) and `GET /api/v1/models` (first-party) return the catalog of models the gateway can route to. The catalog is discovered from provider APIs at runtime, not a static list, so the exact set varies by deployment.

By default the list returns `chat` models. Pass `type` to filter for `image`.

## Request

```bash
curl "https://nito.dev.zprotocol.org/v1/models" \
  -H "Authorization: Bearer z_sk_your_key_here"
```

### Query parameters

| Parameter | Notes |
|---|---|
| `type` | Filter by model type: `chat` (default) or `image`. |

## Response

The envelope is an OpenAI-style list:

```json
{
  "object": "list",
  "data": [
    { "id": "...", "object": "model", "...": "..." }
  ]
}
```

### `apiModel` fields

| Field | Type | Notes |
|---|---|---|
| `id` | string | Logical public model ID. Use this value as `model` in chat/images requests. |
| `object` | string | Always `"model"`. |
| `created` | int | Unix seconds. |
| `owned_by` | string | Owning provider or model owner. |
| `display_name` | string | Human-readable name for UI. |
| `description` | string | Short description. |
| `type` | string | `chat` or `image`. |
| `architecture` | object | `{ input_modalities, output_modalities }`. See below. |
| `capabilities` | object | Per-feature support flags. See below. |
| `context_window` | int | Maximum context length in tokens. |
| `pricing` | object | Unit prices in nanos. See below. |
| `free_tier_eligible` | bool | Whether the model can be used on the free tier. |
| `privacy_tier` | string | Stored privacy tier: `anon`, `private`, or `confidential`. |
| `tee_provider` | string | TEE provider when the tier is `confidential` (`near` or `phala`). |
| `attestation_available` | bool | Whether a TEE attestation can be fetched for this model. |
| `attestation_subject` | string | Attestation subject (NEAR `domain` / Phala `appid`) when applicable. |
| `encrypted_supported` | bool | Whether end-to-end encryption is supported. |

### `architecture`

| Field | Notes |
|---|---|
| `input_modalities` | Accepted input kinds (for example `text`, `image`, `file`). |
| `output_modalities` | Produced output kinds (for example `text`, `image`). |

### `capabilities`

Boolean flags describing what the model supports. The set includes (and is not limited to):

| Flag | Meaning |
|---|---|
| `supports_image_inputs` | Accepts image inputs in chat. |
| `supports_image_generation` | Can generate images. |
| `supports_web_search` | Can use the [web search](/api-reference/server-tools/web-search) server tool. |
| `supports_web_fetch` | Can use the [web fetch](/api-reference/server-tools/web-fetch) server tool. |
| `supports_x_search` | Can use the X/Twitter search tool. |
| `supports_code_interpreter` | Can use the [code interpreter](/api-reference/server-tools/code-interpreter) tool. |
| `supports_reasoning` | Exposes reasoning output. |
| `supports_tee_attestation` | Can produce a TEE attestation. |
| `supports_encrypted` | Supports the Encrypted tier (end-to-end encryption). |

### `pricing`

Per-unit prices are expressed in **nanos** (billionths of a currency unit). Divide any nanos value by `1e9` for the price in the currency; a `*_million_token_nanos` value ÷ `1e9` is the price per 1M tokens (e.g. `1500000000` → `$1.50`).

| Field | Notes |
|---|---|
| `currency_code` | Currency for all amounts, e.g. `USD`. |
| `unit` | Always `nanos`. |
| `input_token_nanos` / `output_token_nanos` | Per-token input / output rate. |
| `cache_read_input_token_nanos` / `cache_write_input_token_nanos` | Cached-input read / write rates. |
| `reasoning_output_token_nanos` | Per-token rate for reasoning output. |
| `image_output_nanos` | Per-image output rate (image models). |
| `web_search_request_nanos` / `web_search_result_nanos` / `web_fetch_request_nanos` | Server-tool per-use rates. |
| `input_million_token_nanos`, `output_million_token_nanos`, `cache_read_million_token_nanos`, `cache_write_million_token_nanos`, `reasoning_million_token_nanos` | Each token rate mirrored per 1M tokens, for convenience. |
| `context_limit_tokens` | The model's context limit. |
| `version` | Opaque price-version identifier. |

## Notes

- The catalog is runtime-discovered and upserted from provider APIs. There is no hardcoded model list, so the available models depend on the deployment's configured providers.
- `pricing` and `context_window` are joined from the billing layer at response time; they are not stored on the underlying model card.
- The `id` you get back is a logical public ID. To request a privacy tier, append `:confidential` or `:encrypted` to it. See [Model groups](/api-reference/models-catalog/model-groups) and [Models](/models/model-groups).

| Endpoint | Status |

| `GET /api/v1/providers` | Not implemented. |

If you need per-endpoint routing or a provider directory, that work is deferred. See [Model groups](/api-reference/models-catalog/model-groups) for the grouping that does exist.
