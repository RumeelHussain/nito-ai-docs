---
title: "Base URL Swap"
---

If your code already uses the OpenAI SDK, you reach Nito by changing three things and nothing else: the base URL, the API key, and the model ID. The request shape, the response shape, streaming, and tool calls all stay the same, because Nito serves the OpenAI-compatible Chat Completions surface.

## The Required Three Changes

1. **Base URL:** point it at your Nito base plus `/v1`.
2. **API key:** use a Nito `z_sk_` key, not an OpenAI key.
3. **Model ID:** use a Nito catalog ID from `GET /v1/models`, not an OpenAI name like `gpt-4o`.

<CodeGroup>

```python Python lines wrap
from openai import OpenAI

client = OpenAI(
    api_key="z_sk_your_key_here",              # your Nito API key
    base_url="https://nito.dev.zprotocol.org/v1",  # Nito's OpenAI-compatible base URL
)

resp = client.chat.completions.create(
    model="x-ai/grok-4.5",                          # a Nito catalog ID, not an OpenAI name
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
)

print(resp.choices[0].message.content)
```

```typescript TypeScript lines wrap
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "z_sk_your_key_here",              // your Nito API key
  baseURL: "https://nito.dev.zprotocol.org/v1",  // Nito's OpenAI-compatible base URL
});

const resp = await client.chat.completions.create({
  model: "x-ai/grok-4.5",                         // a Nito catalog ID
  messages: [
    { role: "user", content: "Say hello in one short sentence." },
  ],
});

console.log(resp.choices[0].message.content);
```

The example model `x-ai/grok-4.5` is illustrative. Do not hard-code it. Call `GET /v1/models` for the live catalog and select by capability at runtime, because models are added and retired over time.

## Why the Base URL Must End in `/v1`

The OpenAI SDK appends the operation path itself. When you call `client.chat.completions.create(...)`, the SDK sends a `POST` to `<base_url>/chat/completions`. So the base URL has to end at the version segment, `/v1`, and not include the operation. If you accidentally set the base URL to `.../v1/chat/completions`, the SDK will build a doubled path and the request will fail.

## Which Surfaces Work This Way

The same base URL swap covers every OpenAI-compatible surface Nito serves:

| **SDK call** | **HTTP surface** |
| --- | --- |
| `client.chat.completions.create(...)` | `POST /v1/chat/completions` |
| `client.embeddings.create(...)` | `POST /v1/embeddings` |
| `client.images.generate(...)` | `POST /v1/images/generations` |
| `client.models.list()` | `GET /v1/models` |

Nito also serves the OpenAI **Responses API** (`client.responses.create(...)`, `POST /v1/responses`) as an OpenAI Responses-compatible endpoint. Chat Completions remains the primary documented text surface. See the [compatibility notes](/sdks/openai-drop-in/compatibility-notes) for the full list of what differs, and the [API Playground](/api-playground) to try the Responses endpoint.

## Reading the Response

The SDK returns a standard `chat.completion` object. `resp.choices[0].message.content` is the assistant's text, and `resp.usage` reports the tokens billed. There is always exactly one choice. `system_fingerprint` is serialized but always `null`, so do not build logic on it.

Streaming works exactly as in the OpenAI SDK: pass `stream=True` (Python) or `stream: true` (TypeScript) and iterate the chunks. Tool calling, prompt caching (`prompt_cache_key`), and reasoning are all supported on this surface.

## Selecting Privacy

Privacy is chosen by the model ID, so you opt into a stronger tier by adding a suffix, with no other code change:

```python wrap
# TEE-backed execution with an attestation you can verify
model="openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support that tier. Use `GET /v1/models` to find models that support attestation. Note that `:encrypted` does not stream and needs extra headers, so it is best done by hand rather than through the SDK. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Common Issues

- **Leaving the OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. See [model mapping](/sdks/openai-drop-in/model-mapping).
- **Forgetting `/v1` on the base URL.** The SDK appends the operation path, so the base URL must end in `/v1`.
- **Using an OpenAI `sk-` key.** Nito keys are prefixed `z_sk_`.
- **Assuming Chat Completions is the only text surface.** It is the primary documented path, but Nito also serves the OpenAI Responses-compatible `/v1/responses` endpoint.
