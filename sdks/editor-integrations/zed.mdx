Zed's assistant supports custom OpenAI-compatible providers. Because Nito exposes an OpenAI-compatible API, you can route Zed's assistant through Nito by filling in that custom-endpoint configuration. You are not selecting a named "Nito" provider. You are using Zed's OpenAI-compatible provider option and pointing it at Nito.

## Values to enter

Wherever Zed accepts OpenAI-compatible provider details, supply:

- **Base URL:** `https://YOUR-NITO-BASE-URL/v1`. The path must end in `/v1`, because OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `grok-4.3`. Do not use an OpenAI name like `gpt-4o`.

## Finding a model ID

Zed will not auto-discover Nito's catalog, so list available models in the custom provider's configuration. Fetch the live list and use one of the returned IDs as the model name:

```bash
curl https://YOUR-NITO-BASE-URL/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

Register the Nito ID as a model name in Zed's provider configuration so it appears in the model selector.

## Choosing a privacy tier

Privacy is selected by the model ID, so no separate Zed setting is required. Append `:tee` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:tee`. Only `:tee` and `:e2e-tee` are valid suffixes, and the model must support the tier. `:e2e-tee` does not stream and needs extra headers, so it is generally not usable from Zed's assistant. Use `GET /v1/models` to find attestation-capable models, and see [Per-call privacy modes](/privacy/per-call-privacy-modes).

## What to expect

- Zed sends standard Chat Completions requests, which Nito serves at `POST /v1/chat/completions`. Streaming and tool calls behave as they do with the OpenAI SDK.
- The OpenAI Responses API and the Anthropic Messages API are not served. Keep Zed on the Chat Completions path.
- If a model is rejected, confirm you used a Nito catalog ID and that the base URL ends in `/v1`.

## Common mistakes

- **Leaving an OpenAI model name in.** Replace `gpt-4o` with a Nito catalog ID from `GET /v1/models`.
- **Dropping `/v1` from the base URL.** The Chat Completions route will not be reached without it.
- **Assuming a built-in Nito provider.** There is none. Use Zed's OpenAI-compatible custom provider.
