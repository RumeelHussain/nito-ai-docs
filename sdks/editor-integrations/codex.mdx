Codex is an OpenAI-oriented coding tool. Since Nito exposes an OpenAI-compatible API, you can route Codex through Nito by overriding its base URL, key, and model. You are not selecting a named "Nito" provider. You are using Codex's OpenAI-compatible configuration and pointing it at Nito.

## Configuration

Codex is configured in `~/.codex/config.toml`. Define Nito as a custom provider and select it at the top level:

```toml
model = "grok-4.3"          # a Nito catalog ID, fetch from GET /v1/models
model_provider = "nito"

[model_providers.nito]
name = "Nito"
base_url = "https://nito.dev.zprotocol.org/v1"   # must end in /v1
env_key = "NITO_API_KEY"    # names the env var Codex reads the key from
wire_api = "chat"           # Chat Completions; use "responses" for the Responses API
```

The key is read from the environment variable named by `env_key`, so export it before launching Codex:

```bash
export NITO_API_KEY="z_sk_your_key_here"
codex
```

<Note>
`wire_api` picks the protocol. Nito serves **both** OpenAI Chat Completions (`/v1/chat/completions`) and a Responses-compatible endpoint (`/v1/responses`), so either `wire_api = "chat"` or `wire_api = "responses"` works. Recent Codex builds default to `"responses"`; set `"chat"` explicitly for the widest compatibility.
</Note>

## Finding a model ID

Codex will not auto-discover Nito's catalog. Fetch the live list and use one of the returned IDs as the model name:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Choosing a privacy tier

Privacy is selected by the model ID, so no separate Codex setting is required. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:confidential`. Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier. `:encrypted` does not stream and needs extra headers, so it is generally not usable from a coding tool's request loop. Use `GET /v1/models` to find attestation-capable models, and see [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Common mistakes

- **Assuming only the Responses API works.** This guide targets Chat Completions; use it for the widest support.
- **Leaving an OpenAI model name in.** Replace `gpt-4o` with a Nito catalog ID from `GET /v1/models`.
- **Dropping `/v1` from the base URL.** The Chat Completions route will not be reached without it.
