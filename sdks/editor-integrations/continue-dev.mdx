Continue.dev lets you add models from custom OpenAI-compatible providers. Because Nito exposes an OpenAI-compatible API, you can add a Nito model to Continue by configuring it as an OpenAI-compatible endpoint. You are not selecting a named "Nito" provider. You are using Continue's OpenAI-compatible model configuration and pointing it at Nito.

## Values to enter

Wherever Continue.dev defines an OpenAI-compatible model, supply:

- **Base URL:** `https://nito.dev.zprotocol.org/v1`. The path must end in `/v1`, because OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `grok-4.3`. Do not use an OpenAI name like `gpt-4o`.

## Finding a model ID

Continue.dev will not auto-discover Nito's catalog. Fetch the live list and use one of the returned IDs as the model name in the configuration:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

Add the Nito ID as the model name so it appears in Continue's model dropdown. You can add several Nito IDs as separate model entries that share the same base URL and key.

## Choosing a privacy tier

Privacy is selected by the model ID, so no separate Continue setting is required. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:confidential`. You can register two entries, one plain and one with the `:confidential` suffix, to switch tiers from the model picker. Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier. `:encrypted` does not stream and needs extra headers, so it is generally not usable from Continue. Use `GET /v1/models` to find attestation-capable models, and see [Per-call privacy modes](/privacy/per-call-privacy-modes).

## What to expect

- Continue.dev sends standard Chat Completions requests, which Nito serves at `POST /v1/chat/completions`. Streaming and tool calls behave as they do with the OpenAI SDK.
- Nito now also serves an OpenAI Responses-compatible endpoint (`/v1/responses`) and an Anthropic Messages-compatible endpoint (`/v1/messages`), but this guide keeps Continue on the Chat Completions path for the widest support.
- If a model entry fails, confirm you used a Nito catalog ID and that the base URL ends in `/v1`.

## Common mistakes

- **Leaving an OpenAI model name in.** Replace `gpt-4o` with a Nito catalog ID from `GET /v1/models`.
- **Dropping `/v1` from the base URL.** The Chat Completions route will not be reached without it.
- **Assuming a built-in Nito provider.** There is none. Use Continue's OpenAI-compatible model configuration.
