Claude Code is an Anthropic-oriented CLI that talks to a Claude model by default. This guide routes it through Nito's **OpenAI-compatible** surface for the widest support. Nito now also serves an Anthropic Messages-compatible endpoint (`POST /v1/messages`); this page targets the OpenAI-compatible path. Use a configuration that points Claude Code at an OpenAI-compatible endpoint.

## Values to enter

Wherever Claude Code accepts an OpenAI-compatible base URL and key, supply:

- **Base URL:** `https://nito.dev.zprotocol.org/v1`. The path must end in `/v1`, because OpenAI-style clients append `/chat/completions` themselves.
- **API key:** your Nito key, which starts with `z_sk_`.
- **Model:** a Nito catalog ID such as `grok-4.3`. Do not use an Anthropic name like `claude-3-5-sonnet` or an OpenAI name like `gpt-4o`.

## This guide uses the OpenAI-compatible path

This page targets OpenAI Chat Completions at `POST /v1/chat/completions` for the widest support. Nito now also serves an Anthropic Messages-compatible endpoint (`/v1/messages`) and its token-counting endpoint (`/v1/messages/count_tokens`). Confirm in the official Claude Code docs whether your version supports an OpenAI-compatible provider before relying on this route.

## Finding a model ID

Fetch the catalog and use one of the returned IDs as the model name:

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

## Choosing a privacy tier

Privacy is selected by the model ID. Append `:confidential` for a TEE-backed run with a verifiable attestation, for example `grok-4.3:confidential`. Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier. `:encrypted` does not stream and needs extra headers, so it is generally not usable from an interactive CLI. Use `GET /v1/models` to find attestation-capable models, and see [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Common mistakes

- **Assuming only the Anthropic Messages path works.** This guide targets the OpenAI-compatible path; use it for the widest support.
- **Leaving a Claude or GPT model name in.** Replace it with a Nito catalog ID from `GET /v1/models`.
- **Dropping `/v1` from the base URL.** The route will not be reached without it.
