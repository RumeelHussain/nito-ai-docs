Claude Code is an Anthropic client — it talks to the **Anthropic Messages API**, not an OpenAI-compatible one. Because Nito now serves an **Anthropic Messages-compatible** endpoint (`POST /v1/messages`), you can point Claude Code at Nito by overriding its Anthropic base URL and key. There is no OpenAI-style configuration for Claude Code.

## Configurations

Claude Code reads its endpoint and credentials from environment variables:

- **`ANTHROPIC_BASE_URL`:** `https://nito.dev.zprotocol.org` — the **host root, without `/v1`**. Claude Code appends `/v1/messages` itself.
- **`ANTHROPIC_AUTH_TOKEN`:** your Nito key (`z_sk_...`), sent as `Authorization: Bearer`. (Use `ANTHROPIC_AUTH_TOKEN`, not `ANTHROPIC_API_KEY`, since Nito issues bearer keys.)
- **`ANTHROPIC_MODEL`:** a Nito catalog ID the Anthropic Messages facade serves, such as `anthropic/claude-sonnet-5`. Use the catalog ID, not a raw Anthropic name like `claude-3-5-sonnet`. Not every catalog model is reachable through this facade — `x-ai/grok-4.5`, for example, is rejected; Claude- and GPT-family catalog IDs work.

### Option A: Environment Variables

```bash wrap
export ANTHROPIC_BASE_URL="https://nito.dev.zprotocol.org"
export ANTHROPIC_AUTH_TOKEN="z_sk_your_key_here"
export ANTHROPIC_MODEL="anthropic/claude-sonnet-5"
claude
```

### Option B: `~/.claude/settings.json`

```json wrap
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://nito.dev.zprotocol.org",
    "ANTHROPIC_AUTH_TOKEN": "z_sk_your_key_here",
    "ANTHROPIC_MODEL": "anthropic/claude-sonnet-5"
  }
}
```

Use a project-level `.claude/settings.json` to scope it to one repo.

## Finding a Model ID

```bash
curl https://nito.dev.zprotocol.org/v1/models \
  -H "Authorization: Bearer z_sk_your_key_here"
```

Use one of the returned IDs as `ANTHROPIC_MODEL`.

## Choosing a Privacy Tier

Privacy is selected by the model ID. Append `:confidential` for a TEE-backed run with a verifiable attestation (`openai/gpt-oss-120b:confidential`). Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier; `:encrypted` is non-streaming and generally not usable from an interactive CLI. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Notes and Caveats

- Nito's Anthropic-compatible surface covers `POST /v1/messages` and `POST /v1/messages/count_tokens`. Features that depend on Anthropic-native infrastructure (e.g. some MCP tooling) may be disabled when pointed at a non-`api.anthropic.com` host.
- `ANTHROPIC_BASE_URL` is the **host root** (no `/v1`); this differs from the OpenAI-compatible integrations, which use `.../v1`.

## Common Issues

- **Using an OpenAI-style base URL.** Claude Code is not an OpenAI client; set `ANTHROPIC_BASE_URL` (host root), not an OpenAI `/v1` URL.
- **Using a raw Anthropic model name.** Replace `claude-3-5-sonnet` with a Nito catalog ID like `anthropic/claude-sonnet-5` from `GET /v1/models`.
- **Picking a model the facade doesn't serve.** `x-ai/grok-4.5` is rejected on `/v1/messages`; use a Claude- or GPT-family catalog ID.
- **Adding `/v1` to `ANTHROPIC_BASE_URL`.** The client appends `/v1/messages` itself.
