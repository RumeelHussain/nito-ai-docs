Many AI coding editors and CLI tools let you bring your own model by configuring a custom OpenAI-compatible provider. Nito exposes an OpenAI-compatible API, so any of those tools can talk to Nito without a Nito-specific plugin. You are not using a named "Nito" provider built into the editor. You are using the editor's generic OpenAI-compatible endpoint setting and pointing it at Nito.

## The shared pattern

Every editor on these pages follows the same three-value setup. Wherever the editor asks for OpenAI-compatible connection details, supply:

- **Base URL:** `https://YOUR-NITO-BASE-URL/v1`. The path must end in `/v1`, because OpenAI-style clients append routes like `/chat/completions` themselves.
- **API key:** a Nito key that starts with `z_sk_`.
- **Model ID:** a Nito catalog ID, such as `grok-4.3`. Do not use an OpenAI name like `gpt-4o`. Fetch the live list with `GET /v1/models`.

## Selecting privacy

Privacy is chosen by the model ID, not by a separate setting. Add a suffix to the model name to opt into a stronger tier:

- `grok-4.3:confidential` runs in a TEE with an attestation you can verify.
- `grok-4.3:encrypted` adds end-to-end encryption, but does not stream and needs extra headers, so it is rarely usable from an editor's chat box.

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support the tier. Use `GET /v1/models` to find attestation-capable models. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## What works and what does not

- The compatible surface is **Chat Completions** (`POST /v1/chat/completions`). Messages, streaming, and tool calls work as in the OpenAI SDK.
- The OpenAI **Responses API** (`/v1/responses`) is now served as an OpenAI Responses-compatible endpoint, though these pages target Chat Completions for the widest tool support.
- The **Anthropic Messages API** (`/v1/messages`) is now served as an Anthropic Messages-compatible endpoint; tools that default to Anthropic-native calls can use it, but these pages target the OpenAI-compatible path for the widest support.

## A note on settings paths

Each editor's exact settings screen, config-file name, and key names change often and are owned by that editor, not by Nito. The per-editor pages below describe the values to enter, not authoritative click-paths. Always confirm the current location against the editor's official documentation.
