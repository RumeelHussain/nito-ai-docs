Pydantic AI agents take a model object, and it supports OpenAI-compatible providers. Nito has no Pydantic AI-specific package; you configure its OpenAI-compatible model for Nito and pass it to your agent, because Nito exposes an OpenAI-compatible API. Your typed outputs, tools, and agent logic stay the same.

## The principle

Configure the OpenAI-compatible model with three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

Hand that model to your agent and the rest of the program is unchanged.

## Representative configuration

```python
# pip install pydantic-ai
from pydantic_ai import Agent
from pydantic_ai.models.openai import OpenAIChatModel
from pydantic_ai.providers.openai import OpenAIProvider

model = OpenAIChatModel(
    "x-ai/grok-4.5",                                    # a Nito catalog ID, fetch from GET /v1/models
    provider=OpenAIProvider(
        api_key="z_sk_your_key_here",              # your Nito API key
        base_url="https://nito.dev.zprotocol.org/v1",  # Nito's OpenAI-compatible base URL
    ),
)

agent = Agent(model)
result = agent.run_sync("Say hello in one short sentence.")
print(result.output)
```

<Note>
Use **`OpenAIChatModel`** (Chat Completions), not `OpenAIResponsesModel`, for a third-party compatible endpoint. The older name `OpenAIModel` is now a deprecated alias of `OpenAIChatModel`.
</Note>

## Choosing a model and privacy

Use a Nito catalog ID for the model name, not an OpenAI name like `gpt-4o`. Fetch the current list from `GET /v1/models`. To run a call inside a TEE with an attestation you can verify, add a suffix to the model ID:

```python
"openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support that tier. Note `:encrypted` does not stream. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## If you reach for the Anthropic model

Pydantic AI's Anthropic model targets the Anthropic Messages API (`/v1/messages`). Nito now serves an Anthropic Messages-compatible endpoint there, but this page uses the OpenAI-compatible model shown above for the widest framework support.

## Common mistakes

- **Leaving an OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Use an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** The client appends `/chat/completions` itself, so the base must end in `/v1`.
- **Selecting the Anthropic model.** This guide targets Chat Completions for the widest support.
