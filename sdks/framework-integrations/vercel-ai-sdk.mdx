The Vercel AI SDK reaches Nito through its OpenAI-compatible provider. There is no Nito-specific provider package. Use the dedicated **`@ai-sdk/openai-compatible`** provider (`createOpenAICompatible`) — the official docs recommend it over `@ai-sdk/openai` for third-party endpoints, because the standard OpenAI provider defaults to the Responses API, which a Chat-Completions endpoint may not implement.

## The principle

Configure the OpenAI-compatible provider with three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

Everything downstream of the provider — your messages, streaming, and tool definitions — follows the OpenAI Chat Completions shape the SDK already produces.

## Representative configuration

```typescript
// npm i ai @ai-sdk/openai-compatible
import { createOpenAICompatible } from "@ai-sdk/openai-compatible";
import { generateText } from "ai";

const nito = createOpenAICompatible({
  name: "nito",
  apiKey: process.env.NITO_API_KEY,              // your Nito key, "z_sk_..."
  baseURL: "https://nito.dev.zprotocol.org/v1",  // Nito's OpenAI-compatible base URL
});

const { text } = await generateText({
  model: nito("x-ai/grok-4.5"),                       // a Nito catalog ID, fetch from GET /v1/models
  prompt: "Say hello in one short sentence.",
});

console.log(text);
```

For streaming, use `streamText` with the same `nito(...)` model. Streaming, tool calling, and structured outputs are all supported (subject to the model supporting them). Note that `:encrypted` does not stream, so use `:confidential` if you need attestation with a streamed response.

<Note>
If you prefer `@ai-sdk/openai` (`createOpenAI({ baseURL, apiKey })`), call the model with `openai.chat("x-ai/grok-4.5")` — the plain `openai("...")` factory targets the Responses API, which the Chat Completions endpoint won't serve.
</Note>

## Choosing a model and privacy

Pass a Nito catalog ID to the provider factory, not an OpenAI name like `gpt-4o`. Fetch the current list from `GET /v1/models`. To run a call inside a TEE with an attestation you can verify, add a suffix to the model ID:

```typescript
model: nito("openai/gpt-oss-120b:confidential"),
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support that tier. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## If you reach for the Anthropic provider

The SDK's Anthropic provider targets the Anthropic Messages API (`/v1/messages`). Nito now serves an Anthropic Messages-compatible endpoint there, but this page uses the OpenAI-compatible provider shown above for the widest framework support.

## Common mistakes

- **Leaving an OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Use an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** The provider appends `/chat/completions` itself, so the base must end in `/v1`.
- **Routing through the Anthropic provider.** This guide targets Chat Completions for the widest support.
