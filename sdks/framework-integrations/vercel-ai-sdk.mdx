The Vercel AI SDK reaches Nito through its OpenAI-compatible provider. There is no Nito-specific provider package. Because Nito exposes an OpenAI-compatible API, you create an OpenAI provider instance configured for Nito and use it the same way you would use any other model in `generateText`, `streamText`, and tool calls.

## The principle

Configure the SDK's OpenAI-compatible provider with three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

Everything downstream of the provider, your messages, streaming, and tool definitions, follows the OpenAI Chat Completions shape the SDK already produces.

## Representative configuration

```typescript
import { createOpenAI } from "@ai-sdk/openai";
import { generateText } from "ai";

const nito = createOpenAI({
  apiKey: "z_sk_your_key_here",              // your Nito API key
  baseURL: "https://YOUR-NITO-BASE-URL/v1",  // Nito's OpenAI-compatible base URL
});

const { text } = await generateText({
  model: nito("grok-4.3"),                   // a Nito catalog ID, fetch from GET /v1/models
  prompt: "Say hello in one short sentence.",
});

console.log(text);
```

For streaming, use `streamText` with the same `nito(...)` model. Note that `:e2e-tee` does not stream, so use `:tee` if you need attestation with a streamed response.

## Choosing a model and privacy

Pass a Nito catalog ID to the provider factory, not an OpenAI name like `gpt-4o`. Fetch the current list from `GET /v1/models`. To run a call inside a TEE with an attestation you can verify, add a suffix to the model ID:

```typescript
model: nito("grok-4.3:tee"),
```

Only `:tee` and `:e2e-tee` are valid suffixes, and the model must support that tier. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## If you reach for the Anthropic provider

The SDK's Anthropic provider targets the Anthropic Messages API (`/v1/messages`), which Nito does not serve; that path returns 404. Use the OpenAI-compatible provider shown above to reach Nito, not the Anthropic provider.

## Common mistakes

- **Leaving an OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Use an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** The provider appends `/chat/completions` itself, so the base must end in `/v1`.
- **Routing through the Anthropic provider.** Nito's compatible surface is Chat Completions, not Messages.
