LiteLLM gives one calling interface across many providers, and it can target any OpenAI-compatible endpoint. Nito has no LiteLLM-specific provider; you route through LiteLLM's OpenAI-compatible path and point it at Nito, because Nito exposes an OpenAI-compatible API. Your calling code stays the same whether you use the Python SDK or the proxy.

## The principle

Supply LiteLLM with three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

LiteLLM forwards the request in OpenAI Chat Completions shape, which is exactly what Nito serves.

## Representative configuration

```python
from litellm import completion

resp = completion(
    model="openai/x-ai/grok-4.5",                       # OpenAI-compatible route; x-ai/grok-4.5 is a Nito catalog ID
    api_key="z_sk_your_key_here",                  # your Nito API key
    api_base="https://nito.dev.zprotocol.org/v1",  # note: api_base (not base_url)
    messages=[{"role": "user", "content": "Say hello in one short sentence."}],
)

print(resp.choices[0].message.content)
```

The `openai/` prefix tells LiteLLM to use its OpenAI-compatible client; the part after it, `x-ai/grok-4.5`, is the Nito catalog ID sent to the gateway. Note the parameter is **`api_base`**, not `base_url`.

**Proxy** — the same three values on a `model_list` entry in `config.yaml`:

```yaml
model_list:
  - model_name: x-ai/grok-4.5
    litellm_params:
      model: openai/x-ai/grok-4.5
      api_base: https://nito.dev.zprotocol.org/v1
      api_key: z_sk_your_key_here
```

Run `litellm --config config.yaml`, then call the proxy with the bare `model_name` (no `openai/` prefix).

## Choosing a model and privacy

Use a Nito catalog ID after the `openai/` route prefix, not an OpenAI name like `gpt-4o`. Fetch the current list from `GET /v1/models` and pick one that supports the tier — private variants are listed as their own IDs ending in `:confidential` / `:encrypted`:

```python
# openai/ = LiteLLM route prefix; openai/gpt-oss-120b:confidential = the Nito catalog ID
model="openai/openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support that tier (e.g. `x-ai/grok-4.5` has no confidential variant). Note `:encrypted` does not stream. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Do not use the Anthropic route

If you call LiteLLM with an `anthropic/` route, it targets the Anthropic Messages API (`/v1/messages`). Nito now serves an Anthropic Messages-compatible endpoint there, but this page routes through the OpenAI-compatible path shown above for the widest framework support.

## Common mistakes

- **Using a real OpenAI model name.** `gpt-4o` is not a Nito catalog ID. Use an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** LiteLLM's OpenAI client appends `/chat/completions` itself, so the base must end in `/v1`.
- **Choosing the `anthropic/` route.** This guide targets Chat Completions for the widest support.
