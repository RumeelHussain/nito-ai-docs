Mastra agents take a model from its built-in **model router**. Nito has no Mastra-specific package; you point the router at Nito's OpenAI-compatible endpoint, because Nito exposes an OpenAI-compatible API. Your agent definitions, tools, and workflows stay the same.

## The principle

Give the agent's `model` the three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

## Representative configuration

Current Mastra uses an **object** model config for a custom endpoint — no AI SDK package needed:

```typescript
// @mastra/core
import { Agent } from "@mastra/core/agent";

export const nitoAgent = new Agent({
  name: "assistant",
  instructions: "Answer concisely.",
  model: {
    url: "https://nito.dev.zprotocol.org/v1",   // base URL, WITH /v1 (not the /chat/completions path)
    id: "nito/grok-4.3",                         // provider/model form; use a Nito catalog ID
    apiKey: process.env.NITO_API_KEY,            // your Nito key, "z_sk_..."
  },
});
```

<Warning>
Use the **object** form above. The bare string `model: "openai/grok-4.3"` routes to the built-in OpenAI provider (hits `api.openai.com` with `OPENAI_API_KEY`) — it will **not** use your custom base URL.
</Warning>

Alternatively, pass an AI SDK v5 provider instance (`npm i ai @ai-sdk/openai`) and use `.chat()` since Nito serves Chat Completions:

```typescript
import { createOpenAI } from "@ai-sdk/openai";
const nito = createOpenAI({ baseURL: "https://nito.dev.zprotocol.org/v1", apiKey: process.env.NITO_API_KEY });
// model: nito.chat("grok-4.3")
```

## Choosing a model and privacy

Use a Nito catalog ID for `id`, not an OpenAI name like `gpt-4o`. Fetch the current list from `GET /v1/models`. To run a call inside a TEE with an attestation you can verify, add a suffix to the model ID:

```typescript
id: "nito/grok-4.3:confidential",
```

Only `:confidential` and `:encrypted` are valid suffixes, and the model must support that tier. Note `:encrypted` does not stream, so prefer `:confidential` for streamed agent responses. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Anthropic-shaped configuration

If you wire Mastra to an Anthropic provider, it targets the Anthropic Messages API (`/v1/messages`). Nito now serves an Anthropic Messages-compatible endpoint there, but this page uses the OpenAI-compatible provider shown above for the widest framework support.

## Common mistakes

- **Leaving an OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Use an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** The provider appends `/chat/completions` itself, so the base must end in `/v1`.
- **Routing through an Anthropic provider.** This guide targets Chat Completions for the widest support.
