LangChain and LangGraph build on the same model interface, so wiring Nito in once gives you both chains and graphs. Nito has no LangChain-specific package. You use LangChain's OpenAI-compatible chat model and redirect it at Nito, because Nito exposes an OpenAI-compatible API. Your prompts, tools, and graph state stay the same.

## The Principle

Configure LangChain's OpenAI chat model with three values:

- the Nito base URL ending in `/v1`,
- a Nito API key (it starts with `z_sk_`),
- a Nito catalog model ID from `GET /v1/models`.

LangGraph nodes that call a model receive this configured model object, so there is nothing graph-specific to change.

## Representative configuration

```python lines wrap
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="x-ai/grok-4.5",                          # a Nito catalog ID, fetch from GET /v1/models
    api_key="z_sk_your_key_here",              # your Nito API key
    base_url="https://nito.dev.zprotocol.org/v1",  # Nito's OpenAI-compatible base URL
)

result = llm.invoke("Say hello in one short sentence.")
print(result.content)
```

The same `llm` object is what you bind tools to (`llm.bind_tools([...])`) and pass into a LangGraph node, so the redirect carries through to graphs unchanged:

```python wrap
from langgraph.prebuilt import create_react_agent
agent = create_react_agent(model=llm, tools=[...])
```

<Note>
  For a non-OpenAI base URL, LangChain leaves streaming token-usage off by default. If you want usage on streamed responses, pass `stream_usage=True` to `ChatOpenAI(...)`.
</Note>

## Choosing a Model and Privacy

Use a Nito catalog ID for `model`, not an OpenAI name like `gpt-4o`. Fetch the list from `GET /v1/models`. To run a call inside a TEE with an attestation you can verify, add a suffix to the model ID and change nothing else:

```python
model="openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes. Note `:encrypted` does not stream, so avoid it for streaming chains. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Reaching for the Anthropic Chat Model

LangChain's Anthropic chat model targets the Anthropic Messages API (`/v1/messages`). Nito now serves an Anthropic Messages-compatible endpoint there, but this page uses the OpenAI-compatible chat model shown above for the widest framework support.

## Common Issues

- **Leaving an OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Replace it with an ID from `GET /v1/models`.
- **Forgetting `/v1` on the base URL.** The client appends `/chat/completions` itself, so the base must end in `/v1`.
- **Pointing at an Anthropic-native class.** This guide targets Chat Completions for the widest support.
