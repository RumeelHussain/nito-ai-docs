Questions about keys, credits, and spend. Each answer links to the page with the full story.

#### What does an API key look like?

It starts with `z_sk_`. Other prefixes (`nito_cli_`, `z_cli_`, `z_mcp_`) belong to the CLI and MCP surfaces, not the HTTP API. See [Authentication](/get-started/authentication).

#### Was I charged for a failed request?

No. Nito places a hold before the call and releases it if the request fails before producing usage. See [Check usage and credits](/get-started/check-usage-and-credits).

#### Where do I see my balance?

Using the API endpoint `GET /api/v1/orgs/{orgId}/billing/balance`. Balance is scoped to the organization, not the individual key, and is returned in nanos (divide by 1,000,000,000 for whole credits).

#### Can I cap what a key spends?

Yes. A key can carry a daily, monthly, or overall spend cap; when the cap is hit, the key is force-expired. Keys have no per-feature scopes; use organization roles to limit who can manage keys.

#### How do I add credits?

Fund through Stripe checkout, or top up an x402 stored balance with on-chain USDC. See [Funding and billing](/api-reference/funding-and-billing).

#### Where do I see usage?

Using the API endpoint `GET /api/v1/orgs/{orgId}/api/overview` returns an aggregated usage summary (requests, tokens, and charge) over a window, with daily and per-model breakdowns. See [Check usage and credits](/get-started/check-usage-and-credits).
