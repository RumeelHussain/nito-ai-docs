Why a request failed, and where the limits are. Each answer links to the page with the full story.

#### Why am I getting 401?

The key is missing, mistyped, revoked, or not sent as `Authorization: Bearer <key>`. API keys start with `z_sk_`.

#### Why am I getting 400 about the model?

The model ID is not in your catalog, or you used an invalid privacy suffix. Only `:confidential` and `:encrypted` are valid. Pull a valid ID from `GET /v1/models`.

#### Why am I getting 402?

The account has no funded balance and the model is not free-tier eligible. Add credits or use a free-tier model.

#### Why am I getting 429?

You hit a rate limit. The response carries a `Retry-After` header telling you how long to wait. There are no `X-RateLimit-*` headers. See [Rate limits](/api-reference/errors-and-rate-limits/rate-limits).

#### Is there a request size cap?

Yes. The chat request body is capped at 4 MiB. The attachments parser has a separate 10 MiB cap. Oversized requests are rejected before any inference runs.

#### Does streaming work with every mode?

No. The `:encrypted` mode is non-streaming only. Use `:confidential` if you need streaming with attestation. See [Streaming](/features/streaming).

#### What is not available yet?

Audio and video input, audio and video generation, and image edits or variations are not available. The `/v1/files` endpoint returns 404 today.
