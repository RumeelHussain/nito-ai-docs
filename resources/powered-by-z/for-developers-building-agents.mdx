Nito gives you a clean, OpenAI-compatible surface for calling models with selectable privacy. If you are building agents and want more than a single inference endpoint, this page points you toward the fuller Z surface underneath.

## Start with what Nito already gives agents

A lot of agent work needs nothing beyond the Nito gateway. Before reaching for anything deeper, note that Nito already supports the building blocks:

- **Tool calling**, so a model can request function calls your code executes. Note the gateway routes tool declarations, calls, and results but does not execute caller-defined functions for you. See [Tool calling](/features/tool-calling).
- **Server-side tools** like web search and web fetch, invoked inline. See [Server-side tools](/features/server-side-tools).
- **Selectable privacy per call** and **attested execution**, so agent workloads can run at the assurance level they need. See [Per-call privacy modes](/privacy/per-call-privacy-modes).
- **Idempotency** for safe retries on billing-sensitive calls. See [Idempotency](/features/idempotency).

For many agents, that surface plus a good loop is enough.

## When you want the fuller Z surface

For advanced agent use cases that go beyond inference, Nito is the entry point to Z, the engine underneath. Think of this as a pointer: Nito is the product surface, and Z is where the broader agent capabilities live. We keep the reference discoverable rather than loud, and we do not restate Z internals here.
