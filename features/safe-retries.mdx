title: "Safe retries"
description: "How to retry a request safely on Nito: failed calls are not billed, and how to guarantee exactly-once at your layer."
---

Networks fail in the worst possible way: your request reaches the gateway, runs, and then the connection drops before you see the response. You do not know whether it worked, so you retry. This page explains how to retry safely on Nito.

## Failed requests are not billed

Nito places a billing hold before each call and captures it only when the call succeeds and produces usage. If a request fails before producing usage — a timeout, a dropped connection, a `5xx` — the hold is released and you are not charged. So retrying a failed request never costs you for the failed attempt. See [How Nito Works](/get-started/how-nito-works) for the hold-and-settle lifecycle.

## Retrying safely

1. **Retry the failures that are actually retryable** — timeouts, dropped connections, `429`, `503` — with backoff. A failed attempt is not billed, so you only pay for the attempt that completes.
2. **Be careful with a call that may have succeeded.** If a request completed on the server but the response was lost in transit, a blind retry will run and bill again. When you cannot tell, check state before retrying.
3. **Guarantee exactly-once at your layer.** For an operation that must not run twice, track completion on your side — for example, a client-generated operation ID you record once the call succeeds and check before retrying.

## Example: retry with backoff

```python
import time, httpx

def call_with_retry(url, payload, headers, attempts=4):
    for i in range(attempts):
        try:
            r = httpx.post(url, json=payload, headers=headers, timeout=60)
            if r.status_code in (429, 500, 502, 503, 504):
                raise httpx.HTTPStatusError("retryable", request=r.request, response=r)
            r.raise_for_status()
            return r.json()
        except (httpx.TransportError, httpx.HTTPStatusError):
            if i == attempts - 1:
                raise
            time.sleep(2 ** i)  # exponential backoff
```

Each failed attempt releases its billing hold, so only the attempt that actually completes is charged.
