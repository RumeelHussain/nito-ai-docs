Quick answers to the questions that come up in the first hour. Each one links to the page with the full story.

## Basics

**Is Nito a model, or a gateway?**
A gateway. Nito sits in front of many models from several providers. You choose the model per request. See [How Nito Works](/get-started/how-nito-works).

**Do I have to use a special SDK?**
No. Point the OpenAI or Anthropic SDK at Nito's base URL and use a Nito model ID. A native SDK exists and is documented in a later pass. See the [OpenAI drop-in](/get-started/quickstart-openai-sdk).

**Which base URL do I use?**
Nito serves an OpenAI-compatible path at `/v1/...` and a first-party path at `/api/v1/...`. For chat they behave identically. The production host is being finalized; this is marked as a TODO across the quickstarts.

**How do I find available models?**
Call `GET /v1/models`. The catalog is discovered from providers and kept current, so the live list is authoritative. Do not hard-code model names.

## Privacy

**Does every call run in a TEE?**
No. TEE execution is the highest tier and is selected per call by adding `:tee` to the model ID (and the model must support it). Most calls use lighter modes. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

**How do I make a call private?**
Add a suffix to the model ID: `:tee` for TEE-backed execution with attestation, or `:e2e-tee` for end-to-end encrypted TEE. Ordinary and private modes need no suffix. Only `:tee` and `:e2e-tee` are valid.

**Can I verify the privacy claim myself?**
Yes, for TEE tiers. Fetch an attestation from `GET /api/v1/tee/attestation` and verify it independently. The gateway performs a baseline verification and returns the raw evidence so you can do full verification client-side. See [TEE attestation](/privacy/tee-attestation).

**What does Nito keep about my prompts?**
By default, metadata only. Prompt and completion contents are treated as request-local and are not persisted by the gateway. See [Zero Data Retention](/privacy/zero-data-retention).

## Keys and billing

**What does an API key look like?**
It starts with `z_sk_`. Other prefixes (`nito_cli_`, `z_cli_`, `z_mcp_`) are for the CLI and MCP surfaces, not the HTTP API. See [Authentication](/get-started/authentication).

**Can I limit what a key can spend?**
Yes. A key can carry a daily, monthly, or overall spend cap; when the cap is hit, the key is automatically expired. Keys do not have per-feature scopes; to limit management, use organization roles.

**Was I charged for a failed request?**
No. Nito places a hold before the call and releases it if the request fails before producing usage. See [Check usage and credits](/get-started/check-usage-and-credits).

**Where do I see my balance?**
`GET /api/v1/orgs/{orgId}/billing/balance`. Balance is per organization, returned in nanos.

## Errors

**Why am I getting 401?**
The key is missing, mistyped, revoked, or not sent as `Authorization: Bearer <key>`.

**Why am I getting 400 about the model?**
The model ID is not in your catalog, or you used an invalid privacy suffix. Only `:tee` and `:e2e-tee` are valid suffixes. Pull a valid ID from `GET /v1/models`.

**Why am I getting 402?**
The account has no funded balance and the model is not free-tier eligible. Add credits or use a free-tier model.

**Why am I getting 429?**
You hit a rate limit. The response includes a `Retry-After` header telling you how long to wait. See [Rate limits](/resources/rate-limits).
