---
title: "OpenAI SDK Drop-in"
description: "Point the OpenAI SDK at Nito by changing the base URL. Your existing code keeps working."
---

If your code already uses the OpenAI SDK, you do not need to rewrite it. Nito exposes an OpenAI-compatible endpoint, so you change three things: the base URL, the API key, and the model ID. Messages, streaming, tool calls, and the response shape stay the same.

<CodeGroup>

```python Python wrap
from openai import OpenAI

client = OpenAI(
    api_key="z_sk_your_key_here",          # your Nito API key
    base_url="https://nito.dev.zprotocol.org/v1",  # Nito's OpenAI-compatible base URL
)

resp = client.chat.completions.create(
    model="x-ai/grok-4.5",                      # a Nito catalog ID, not an OpenAI name
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
)

print(resp.choices[0].message.content)
```

```typescript TypeScript wrap
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "z_sk_your_key_here",              // your Nito API key
  baseURL: "https://nito.dev.zprotocol.org/v1",  // Nito's OpenAI-compatible base URL
});

const resp = await client.chat.completions.create({
  model: "x-ai/grok-4.5",                         // a Nito catalog ID
  messages: [
    { role: "user", content: "Say hello in one short sentence." },
  ],
});

console.log(resp.choices[0].message.content);
```

The base URL must end in `/v1`, because the SDK appends paths like `/chat/completions` itself. Use a Nito catalog ID for `model`, not an OpenAI name like `gpt-4o`.

## Reading the Response

The SDK sends a POST to `/v1/chat/completions` and returns a standard `chat.completion` object. `resp.choices[0].message.content` is the assistant's text, and `resp.usage` reports tokens billed. Streaming works exactly as in the OpenAI SDK: pass `stream=True` (Python) or `stream: true` (TypeScript) and iterate the chunks.

## Selecting Privacy

Because privacy is chosen by the model ID, you opt into a stronger tier by adding a suffix to the model name, with no other code change:

```python
# TEE-backed execution with an attestation you can verify
model="openai/gpt-oss-120b:confidential"
```

Only `:confidential` and `:encrypted` are valid suffixes, and a model must support that tier. Use `GET /v1/models` to find models that support attestation. See [Per-call privacy modes](/privacy/per-call-privacy-modes).

## Common Issues

- **Leaving the OpenAI model name in.** `gpt-4o` is not a Nito catalog ID. Replace it with a Nito model ID from `GET /v1/models`. Model mapping is covered in [OpenAI SDK drop-in compatibility](/sdks/openai-drop-in/model-mapping).
- **Forgetting `/v1` on the base URL.** The OpenAI SDK appends paths like `/chat/completions`, so the base URL must end in `/v1`.
- **Assuming the Responses API is unavailable.** Nito now serves an OpenAI Responses-compatible endpoint at `POST /v1/responses`. Chat Completions remains the primary, fully supported surface and is what this page uses.
- **Streaming with Encrypted.** The `:encrypted` mode does not stream. Use `:confidential` if you need streaming with attestation.

## Important Notes

- **Rate limits:** standard per-key and per-org limits apply. A 429 includes a `Retry-After` header. See [Rate limits](/api-reference/errors-and-rate-limits/rate-limits).
- **Retention:** metadata-only by default. See [Zero Data Retention](/privacy/zero-data-retention).
- **Privacy-mode applicability:** ordinary, private, and `:confidential` work with the OpenAI SDK. `:encrypted` requires extra headers and is best done via curl or the native SDK.
