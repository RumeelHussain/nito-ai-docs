If your codebase is built around the Anthropic SDK, this page shows how to reach Nito with the least change. Nito's most widely used compatible surface is the OpenAI Chat Completions shape, and routing Anthropic-based code through Chat Completions is a supported, working path. Nito now also serves an Anthropic Messages-compatible endpoint at `POST /v1/messages` (and `POST /v1/messages/count_tokens`), so the native shape reaches the gateway too.

<Note>
  Nito now serves an Anthropic Messages-compatible endpoint at `POST /v1/messages` (and `POST /v1/messages/count_tokens`): the gateway validates the Anthropic Messages request shape and maps it into its canonical chat contracts. You can try it in the [API Playground](/api-playground). The OpenAI-compatible Chat Completions route described below also remains fully supported.
</Note>

## The recommended path today

Point your client at Nito's OpenAI-compatible endpoint and use the Chat Completions shape. Most Anthropic-based applications keep a thin abstraction over the model call, so the change is usually confined to one client setup file.

<CodeGroup>

```python Python
from openai import OpenAI

# Route Anthropic-style code through Nito's OpenAI-compatible surface.
client = OpenAI(
    api_key="z_sk_your_key_here",
    base_url="https://nito.dev.zprotocol.org/v1",
)

resp = client.chat.completions.create(
    model="x-ai/grok-4.5",   # a Nito catalog ID from GET /v1/models
    messages=[
        {"role": "system", "content": "You are a concise assistant."},
        {"role": "user", "content": "Say hello in one short sentence."},
    ],
)

print(resp.choices[0].message.content)
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "z_sk_your_key_here",
  baseURL: "https://nito.dev.zprotocol.org/v1",
});

const resp = await client.chat.completions.create({
  model: "x-ai/grok-4.5",
  messages: [
    { role: "system", content: "You are a concise assistant." },
    { role: "user", content: "Say hello in one short sentence." },
  ],
});

console.log(resp.choices[0].message.content);
```

## Mapping notes

- **System prompts.** Anthropic takes a top-level `system` string; the Chat Completions shape uses a message with `role: "system"`. Move the system text into the messages array as shown above.
- **Roles.** Nito accepts `system`, `user`, `assistant`, `tool`, and `developer` roles on the inference layer.
- **Model names.** Anthropic model names are not Nito IDs. Use a Nito catalog ID from `GET /v1/models`.
- **Privacy.** As everywhere on Nito, add `:confidential` or `:encrypted` to the model ID to raise the privacy tier.

## Common mistakes

- **Assuming `/v1/messages` is unavailable.** It is now served as an Anthropic Messages-compatible endpoint. This page routes through `/v1/chat/completions`, which also works.
- **Keeping a top-level `system` field.** It will not be read on the Chat Completions surface. Convert it to a `system` message.
- **Carrying over an Anthropic model name.** Replace it with a Nito ID.

## Notes

- Rate limits: per-key and per-org; 429 carries `Retry-After`. See [Rate limits](/api-reference/errors-and-rate-limits/rate-limits).
- Retention: metadata-only by default. See [Zero Data Retention](/privacy/zero-data-retention).
- Privacy-mode applicability: ordinary, private, and `:confidential`. `:encrypted` is non-streaming and needs extra headers.
